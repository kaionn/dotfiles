# Obsidian ファイル整理（タグベース）

00_inbox/temp/ ディレクトリ内の Markdown ファイルをタグに基づいて適切なディレクトリに振り分けます。

**作業ディレクトリ:** `/Users/aucks/Library/Mobile Documents/iCloud~md~obsidian/Documents/Main`

以下のすべてのパスは上記ディレクトリからの相対パスです。

以下の手順で実行してください：

## Step 1: 対象ファイルの確認

`00_inbox/temp/` ディレクトリ内のすべての Markdown ファイルをリストアップし、各ファイルのタグを確認します。

```bash
ls -la 00_inbox/temp/*.md
```

## Step 2: 各ファイルのタグ分析

各ファイルのフロントマターを読み取り、タグを確認します。以下の分類ルールに従ってファイルを振り分けます：

### 分類ルール：

- **`daily` タグ** → `00_inbox/daily/` へ移動
- **`note` タグ** → `02_working/` へ移動
- **タグなし or 該当なし** → そのまま残す（報告のみ）

## Step 3: ファイルの移動

各ファイルを適切なディレクトリに移動します：

```bash
# daily タグの場合
mv "00_inbox/temp/[ファイル名].md" "00_inbox/daily/[ファイル名].md"

# note タグの場合
mv "00_inbox/temp/[ファイル名].md" "02_working/[ファイル名].md"
```

**注意事項：**
- 複数のタグが含まれる場合、優先順位は `daily` > `note` とする
- ファイル名の重複がある場合は、ユーザーに確認する
- 移動先ディレクトリが存在しない場合は作成する

## Step 4: 移動結果の確認

移動後、以下を確認します：

- 各ディレクトリに正しくファイルが配置されていること
- `00_inbox/temp/` に残っているファイル（移動対象外）があればリスト表示

## Step 5: 完了報告

以下の内容をレポートします：

- **移動したファイル数の合計**
- **`00_inbox/daily/` への移動**: ファイル数とファイル名
- **`02_working/` への移動**: ファイル数とファイル名
- **移動対象外**: ファイル数とファイル名（あれば）

---

**実行時の注意：**

- ファイル名にスペースや特殊文字が含まれる場合は適切にクォート
- 移動前に対象ファイルを確認し、問題がないか確認すること
- ファイルの内容は変更しない（移動のみ）
